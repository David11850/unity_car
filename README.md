# unity_car
## 系统总体架构 (4 层结构)
1. 仿真环境与物理层 (Unity 端)
   负责构建高保真的三维校园、模拟小车的物理动力学、以及生成虚拟传感器数据（相当于真实的物理世界）。

2. 通信中间件层 (ROS-TCP-Connector)
    负责打通 Unity（C#）和 ROS（Python/C++）的数据传输通道，实现高频、低延迟的消息订阅与发布。

3. 单车智能控制层 (ROS 端 / NavStack)S
    负责接收虚拟传感器数据，进行建图、定位、全局路径规划和局部动态避障。

4. 顶层多车调度与订单业务层 (中央调度服务器 / Python)
    负责模拟外卖平台的订单分发系统，统筹所有小车的位置，进行任务分配和路口防死锁调度。
