# unity_car

## 整体架构
前后端分离，unity只负责前端部分，计算全部给后端去做(所有路径规划、避障、调度计算都在 C++ 后端)

  ┌─────────────────────────────────────────────────────────┐
  │                     C++ 后端 (大脑)                      │
  │  ┌──────────┐  ┌──────────┐  ┌──────────────────────┐   │
  │  │ epoll    │  │ 调度算法  │  │ 路径规划 (A*/Dijkstra)│   │
  │  │ 网络I/O   │  │ 订单分配  │  │ 避障 / 多车协调       │   │
  │  └────┬─────┘  └────┬─────┘  └──────────┬───────────┘   │
  │       │              │                    │             │
  │       └──────────────┴────────────────────┘             │
  │                         │                               │
  │              每帧/每tick 计算控制指令                      │
  │              发送: CMD_VEL,carId,vx,vy,vz,yawRate        │
  │              发送: CMD_POS,carId,x,y,z                   │
  └─────────────────────────────┬───────────────────────────┘
                                │ TCP
                                ▼
  ┌─────────────────────────────────────────────────────────┐
  │                   Unity 前端 (躯体)                      │
  │  ┌──────────────┐    ┌──────────────┐                  │
  │  │ CarTcpClient │───▶│ 小车物理模型  │                   │
  │  │ (网络接收)    │    │ (Rigidbody/  │                   │
  │  │              │    │  WheelCollider│                 │
  │  └──────┬───────┘    └──────────────┘                  │
  │         │                                              │
  │         │ 每帧/固定频率                                  │
  │         ▼                                              │
  │  上报: STAT,carId,x,y,z,vx,vz,state                     │
  └─────────────────────────────────────────────────────────┘
